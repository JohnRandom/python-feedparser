Relative Link Resolution

Many feed elements and attributes are URIs. Universal Feed Parser resolves relative URIs according to the XML:Base specification.

These feed elements are treated as URIs, and resolved if they are relative:

  • feed.link
  • feed.links[i].href
  • feed.generator_detail.url
  • feed.id
  • feed.image.url
  • feed.image.link
  • feed.textinput.link
  • feed.author_detail.url
  • feed.publisher_detail.url
  • feed.contributors[i].url
  • feed.docs
  • feed.license
  • entries[i].link
  • entries[i].links[j].href
  • entries[i].id
  • entries[i].author_detail.url
  • entries[i].publisher_detail.url
  • entries[i].contributors[j].url
  • entries[i].enclosures[j].url
  • entries[i].source.url
  • entries[i].comments
  • entries[i].license

In addition, several elements may contain HTML or XHTML markup. Certain elements and attributes in HTML can be relative URIs, and Universal Feed Parser will resolve these URIs according to the same rules as the feed elements listed above.

  • <a href="...">
  • <applet codebase="...">
  • <area href="...">
  • <blockquote cite="...">
  • <body background="...">
  • <del cite="...">
  • <form action="...">
  • <frame longdesc="...">
  • <frame src="...">
  • <iframe longdesc="...">
  • <iframe src="...">
  • <head profile="...">
  • <img longdesc="...">
  • <img src="...">
  • <img usemap="...">
  • <input src="...">
  • <input usemap="...">
  • <ins cite="...">
  • <link href="...">
  • <object classid="...">
  • <object codebase="...">
  • <object data="...">
  • <object usemap="...">
  • <q cite="...">
  • <script src="...">
